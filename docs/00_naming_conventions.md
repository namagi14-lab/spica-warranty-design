# 00 命名規則

## 1. 構成階層

```
HOST PC（全工程・全ライン共通）
└── LINE（ライン）
    └── 工程
        └── CELL（セル）
            └── ZONE（ゾーン）
```

---

## 2. ライン (LINE)

**フォーマット**: `LINE{SEQ}`

| 例 | 説明 |
|----|------|
| `LINE1` | 1番目のライン |
| `LINE2` | 2番目のライン |

---

## 3. 工程

**フォーマット**: `{工程種別}{SEQ}`

- 工程種別の英字は工程の種類を表す（例: `A` = 仕上げ工程）
- 連番はライン内で一意

| 例 | 説明 |
|----|------|
| `A1` | 仕上げ工程 1番目 |
| `A2` | 仕上げ工程 2番目 |
| `A3` | 仕上げ工程 3番目 |

---

## 4. セル (CELL)

**フォーマット**: `CELL{SEQ:02}`

- 工程内で連番。ライン・工程・セル名の組み合わせで一意になる

| 例 | 説明 |
|----|------|
| `CELL01` | 工程内の1番目のセル |
| `CELL02` | 工程内の2番目のセル |

---

## 5. ゾーン (ZONE)

**フォーマット**: `ZONE{SEQ:02}`

- セル内で連番。ライン・工程・セル・ゾーン名の組み合わせで一意になる

| 例 | 説明 |
|----|------|
| `ZONE01` | セル内の1番目のゾーン |
| `ZONE02` | セル内の2番目のゾーン |

---

## 6. デバイス

### 6-1. セルに紐づくデバイス

**フォーマット**: `{LINE}_{工程}_{CELL}_{DEVICE_TYPE}{SEQ:02}`

| デバイス | コード | 例 (LINE1・A1工程・CELL01・1台目) | 例 (将来2台目) |
|----------|--------|----------------------------------|---------------|
| タブレット | `TAB` | `LINE1_A1_CELL01_TAB01` | `LINE1_A1_CELL01_TAB02` |
| MiniPC | `MPC` | `LINE1_A1_CELL01_MPC01` | `LINE1_A1_CELL01_MPC02` |

- 異なるLINEまたは工程の同名CELLは別デバイスとして区別される
  - 例: `LINE1_A1_CELL01_TAB01` ≠ `LINE2_A1_CELL01_TAB01`
- 連番は2桁ゼロパディング（01〜99）で将来の複数台に対応

### 6-2. 全ライン・全工程共通デバイス

**フォーマット**: `{DEVICE_TYPE}{SEQ:02}`

| デバイス | コード | 例 | 例 (将来2台目) |
|----------|--------|----|---------------|
| Host PC | `HPC` | `HPC01` | `HPC02` |

- Host PC は全ライン・全工程をまたいで管理するため、LINE・工程・セル名を含めない

---

## 7. 命名早見表

```
LINE:         LINE1, LINE2, ...
工程:         A1, A2, A3, ...          （LINEの中で連番）
CELL:         CELL01, CELL02, ...      （工程の中で連番）
ZONE:         ZONE01, ZONE02, ...      （セルの中で連番）

タブレット:   {LINE}_{工程}_{CELL}_TAB{SEQ}   例: LINE1_A1_CELL01_TAB01
MiniPC:       {LINE}_{工程}_{CELL}_MPC{SEQ}   例: LINE1_A1_CELL01_MPC01
Host PC:      HPC{SEQ}                          例: HPC01
```
