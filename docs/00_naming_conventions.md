# 00 命名規則

## 1. 工程 (Process)

**フォーマット**: `{英字}{数字}`

| 例 | 説明 |
|----|------|
| `A1` | A工程 第1ライン |
| `A2` | A工程 第2ライン |

---

## 2. セル (CELL)

**フォーマット**: `CELL{SEQ:02}`

- 工程名とセル名の組み合わせで一意になる（セル名単体では一意でない）

| 例 | 説明 |
|----|------|
| `CELL01` | 1番目のセル |
| `CELL02` | 2番目のセル |

---

## 3. ゾーン (ZONE)

**フォーマット**: `ZONE{SEQ:02}`

- セルの中に複数のZONEが存在する
- 工程名・セル名・ゾーン名の組み合わせで一意になる

| 例 | 説明 |
|----|------|
| `ZONE01` | セル内の1番目のゾーン |
| `ZONE02` | セル内の2番目のゾーン |

---

## 4. デバイス

### 4-1. セルに紐づくデバイス

**フォーマット**: `{工程}_{CELL}_{DEVICE_TYPE}{SEQ:02}`

| デバイス | コード | 例 (A1工程・CELL01・1台目) | 例 (将来2台目) |
|----------|--------|--------------------------|---------------|
| タブレット | `TAB` | `A1_CELL01_TAB01` | `A1_CELL01_TAB02` |
| MiniPC | `MPC` | `A1_CELL01_MPC01` | `A1_CELL01_MPC02` |

- 同じ `CELL01` でも工程が異なれば別デバイス（例: `A1_CELL01_TAB01` ≠ `A2_CELL01_TAB01`）
- 連番は2桁ゼロパディング（01, 02 ... 99）で拡張に対応

### 4-2. 全工程共通デバイス

**フォーマット**: `{DEVICE_TYPE}{SEQ:02}`

| デバイス | コード | 例 | 例 (将来2台目) |
|----------|--------|----|---------------|
| Host PC | `HPC` | `HPC01` | `HPC02` |

- Host PC は全工程をまたいで管理するため、工程名・セル名を含めない

---

## 5. 命名早見表

```
工程:    A1, A2, B1, B2, ...
CELL:    CELL01, CELL02, ...         （工程内で連番）
ZONE:    ZONE01, ZONE02, ...         （セル内で連番）

タブレット:   {工程}_{CELL}_TAB{SEQ}   例: A1_CELL01_TAB01
MiniPC:       {工程}_{CELL}_MPC{SEQ}   例: A1_CELL01_MPC01
Host PC:      HPC{SEQ}                  例: HPC01
```
